WooCommerce POS - Orders & Cash Drawer Management Guide

Processing POS Orders

Overview

POS users can manage complete sales activity including adding products to cart, managing customer details, holding carts, and processing payments efficiently.

Sales Management Capabilities

Core Functions:

  • Adding products to cart
  • Adding customer details
  • Holding cart for later
  • Processing payments
  • Generating receipts
  • Managing discounts and coupons

Adding Products to Cart

Methods to Add Products:

  1. Barcode Scanning: Scan product barcode using barcode reader
  2. Manual Search: Search by product name or SKU
  3. Category Browse: Navigate through product categories
  4. Custom Products: Add products not available on website but available in physical store

Cart Operations:

  • Adjust product quantities
  • Remove products from cart
  • View cart totals
  • Apply discounts

Customer Management

Adding Customer Details:

  1. Search for existing customer by name or email
  2. Add new customer on-the-fly
  3. Use default customer for walk-in sales
  4. Update customer information as needed

Payment Processing

Available Payment Options:

  • Cash Payment: Primary payment method
  • Card Payment: Credit/debit card transactions
  • Custom Payment Methods: Additional configured payment options

Checkout Process:

  1. Add all products to cart
  2. Select or add customer
  3. Apply discounts or coupons if applicable
  4. Choose payment method(s)
  5. Process payment
  6. Generate receipt

Split Payment:

  • Customers can go for split payment between multiple methods
  • Multiple payment methods recorded in POS
  • Example: Part cash, part card
  • System tracks each payment method separately

Discounts & Coupons

Coupon Application:

  • POS agent can apply coupon to cart at time of checkout
  • Enter coupon code manually
  • System validates and applies discount
  • View discount breakdown

Customer-Based Discount:

  • Applicable for few products
  • Automatic application based on customer group
  • Special pricing for loyalty members

Price Editing:

  • POS user can edit price of items in cart
  • Manual price adjustments when authorized
  • Override default pricing

Unit Price Calculation

Weight-Based Pricing:

  • Sales agent enters product weight
  • System automatically calculates price
  • Real-time price updates
  • Modify weight as needed

Receipt Generation

Invoice Printing:

  • User can print invoice for order
  • Automatic or manual printing
  • Customizable receipt templates
  • Thermal printer support

Hold Cart & Resume

Hold Cart Functionality

Overview:

Cart can be put on hold for future processing, allowing POS users to manage multiple customers efficiently.

Putting Cart on Hold

Step 1: Initiate Hold

  • User puts cart "on hold" to process after some time
  • Click Hold Cart option in panel
  • Available in cart actions

Step 2: Add Order Note

  • Order Note tab opens after POS user puts cart on hold
  • Enter order note for reference
  • Add customer name or special instructions
  • Save hold cart

Viewing Hold Carts

Access Held Carts:

Method 1: Quick Access

  • View hold carts in top right corner of cart
  • Click hold cart icon
  • See list of all held carts

Method 2: Through Orders Menu

  • Navigate through POS > Order > Hold Sale
  • View all held carts
  • Search and filter options

Processing Held Carts

Resume Processing:

  1. Select held cart from list
  2. Cart loads with all previous items
  3. Continue with checkout process
  4. Complete transaction

Hold Sale Management:

  • Cart put on hold can be processed later
  • No time limit on held carts
  • Admin can view hold carts in Orders > Hold Sale
  • Track multiple held carts simultaneously

Cash Drawer Opening/Closing

Opening Cash Drawer

Configuration:

  • Opening amount drawer status: After enabling option, popup shows for POS to enter opening amount of cash drawer
  • Configure in admin settings
  • Enable/disable opening amount requirement

Opening Process:

  1. POS agent logs in
  2. Opening cash drawer popup appears
  3. POS agent enters cash drawer amount (example: $1000)
  4. Amount required for particular day
  5. Submit and begin sales

Close Counter Tab

Accessing Close Counter:

  • Available under Cashier section
  • End-of-day drawer reconciliation
  • Complete cash management

Close Counter Components:

  1. Drawer Amount Details
  2. Counted Drawer Amount
  3. Closing Drawer Details

Drawer Amount Details

Display Information:

  • Opening Amount: Initial cash in drawer
  • Today's Sales: Total sales for the day
  • Expected Amount in Drawer: Opening + Sales

Example:

Opening Amount: $1,000
Today's Sales: $2,500
Expected Amount: $3,500

Counted Drawer Amount

Cash Counting:

  • Cash amount collected for particular day
  • Manual count entry
  • Remarks option available

Recording Process:

  1. Count physical cash in drawer
  2. Enter counted amount
  3. Add remarks if needed
  4. Note any discrepancies

Closing Drawer Details

Display Information:

  • Displays closing balance details for particular day
  • Shows difference between expected and actual
  • Difference calculation between opening and closing amount viewed under Closing Drawer Details

Variance Tracking:

  • Over/short amounts highlighted
  • Requires explanation for large variances
  • Historical tracking of drawer accuracy

Drawer Close Email

Email Functionality:

  • Drawer Close Email functionality for Admins
  • Automatic email notifications
  • Drawer closure reports sent to management

Email Contents:

  • Opening amount
  • Total sales
  • Expected amount
  • Counted amount
  • Difference/variance
  • Timestamp and agent name

Auto Drawer Close

Configuration:

  • Enable/Disable option with drawer time setting
  • Set automatic closure time
  • Configure in admin settings

Auto Close Features:

  • Drawer closure management: Admin effectively manages drawer closures ensuring process completed properly
  • Scheduled automatic closure
  • Sends reminders before auto-close
  • Forces reconciliation at set time

Drawer Management & Reports

Admin Controls:

  • Email reports: Admin can send email report to user using bulk action or individual report options
  • Drawer list: Admin reviews drawer closure details to ensure everything properly completed
  • Track agent performance
  • Identify patterns and issues

Refunds & Returns

WooCommerce Order Refunds in POS

Refund Visibility:

  • WooCommerce website order refund details visible at POS
  • Real-time refund synchronization
  • Track refund status

Initiating Refunds

Refund Process:

  1. If refund initiates for WooCommerce website order by admin
  2. Refund details visible at POS for particular order
  3. POS agents can view refund information
  4. Process return at POS terminal

Order Management for Refunds

Viewing Order Details:

  1. Admin can view order details for specific order
  2. Order editing: Click on order (example: #211) to view order details
  3. View refund status and amount
  4. Track refund method

Refund Information Display:

  • Original order details
  • Refund amount
  • Refund date and time
  • Refund reason
  • Refund method (original payment method)

Synchronizing Offline Orders

Offline Sales Management

Overview:

  • Offline sales orders processed in offline mode seen under POS > Order > Offline Sales
  • System stores orders locally when offline
  • Automatic synchronization when online

Manual Synchronization

Sync Order Button:

  1. Navigate to POS > Order > Offline Sales
  2. Click Sync Order button
  3. All offline orders synced to online store
  4. Verify synchronization completion

Auto Sync Configuration

Automatic Synchronization:

  • Auto Sync Offline Orders: Enable option for offline orders to sync automatically when creating any order online
  • Configure in settings
  • No manual intervention needed

Important Note:

  • Offline functionality only works with Master Stock Inventory
  • Centralized inventory required
  • Configure inventory type in settings

Synchronization Process

How It Works:

  1. POS agents can synchronize data with online store whenever going online
  2. Admin can click offline sync button to synchronize offline orders
  3. Point Of Sale systems work both online and offline with synchronization feature
  4. Offline data sync: All sales and customer data of offline store sync with online store whenever going online

Synchronized Data:

  • Orders and transactions
  • Customer information
  • Stock updates
  • Payment records

Payment Reconciliation

Today Cash Overview

Today Cash Option Display:

  • Cash opening balance
  • Current cash balance
  • Current card balance

Information Available:

Payment details visible to POS agent under Today Cash option:

  • Order ID
  • Time
  • Orders
  • Payment mode placed

Payment Recording

System Tracking:

  • POS records payment methods used
  • Individual transaction tracking
  • Payment method breakdown

Tracked Information:

  • Cash transactions
  • Card transactions
  • Custom payment methods
  • Split payments

Cash Drawer Reconciliation

Daily Tracking:

  • Cash drawer management: Today's sales and expected amount in Drawer tracked
  • Counted Drawer Amount: Cash amount collected for particular day
  • Closing balance: Difference between opening and closing amount tracked

Reconciliation Report:

Shows:

  • Opening balance
  • Total sales
  • Expected closing
  • Actual closing
  • Variance

Payment Mode Tracking

Detailed Records:

  • Orders and payment mode placed visible
  • Payment method distribution
  • Transaction timeline
  • Customer payment preferences

Order History & Tracking

Viewing Orders

Admin Access:

Navigate to POS > Orders to:

  • View list of orders
  • Search and filter orders
  • Access detailed order information

Order Details

Viewing Specific Orders:

  1. Admin can view Order Details for specific order
  2. Click on order number to open details
  3. View complete order information

Order Information Includes:

  • Products purchased
  • Quantities
  • Prices and discounts
  • Customer information
  • Payment method
  • Order status
  • Timestamps

Sale History

Accessing Sale History:

Navigate to POS > Order > Sale History

Sale History Features:

  • User can see all sales recorded in system
  • View order history
  • Generate invoice for orders
  • Export sales data

Order List Display:

POS user can view:

  • Order ID
  • Time
  • Orders
  • Payment mode

WooCommerce Integration

Order Synchronization:

  • WooCommerce website orders listed on POS
  • Seamless integration between channels
  • Real-time order updates

Channel Identification:

  • Orders placed at WooCommerce website show channel name as WooCommerce
  • POS orders show channel name as POS
  • Easy identification of order source

Custom Order Prefix

Order Numbering:

  • Admin can add custom order prefix
  • Example: #WOO_470 where 470 is order ID and WOO_ is prefix
  • Helps organize orders by channel or location
  • Configure in admin settings

Sale History in Cashier Section

Cashier Dashboard:

Sale History Option under Cashier section shows:

  • Cash sale listed for particular date
  • Card sale listed for particular date
  • Total sale display: Total sales after adding cash and card sale

Data Visibility:

  • Data visible when cash drawer closed in "Close Counter" section
  • Historical data accessible
  • Filter by date range
  • Export capabilities

Order Search & Filtering

Search Options:

  • Search by order ID
  • Search by customer name
  • Search by date range
  • Filter by payment method
  • Filter by order status

Advanced Filtering:

  • Multiple filter combinations
  • Custom date ranges
  • Agent-specific orders
  • Outlet-specific orders
