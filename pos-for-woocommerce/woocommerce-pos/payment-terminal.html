WooCommerce POS - Payment Terminal Integration

Supported Payment Terminals

1. Sunmi Terminal

  • Type: Credit/Debit card payment terminal
  • Integration: Direct integration with WooCommerce POS
  • Functionality: Process card payments at physical store locations
  • Recording: Payment transactions recorded in POS system

2. Square Payment Terminal

  • Plugin Required: WooCommerce POS Square Payment Terminal plugin
  • Card Support: Credit and Debit cards
  • Usage: Process payments directly at POS outlet
  • Integration: Seamless connection with WooCommerce POS system

Payment Methods Support

Multiple Payment Options

The admin can configure various payment methods for recording purposes at checkout:

Available Payment Methods

  • Cash Payment
  • Card Payment (via terminals)
  • PayPal
  • Sunmi Terminal (Credit/Debit)
  • Square Terminal (Credit/Debit)
  • Other Custom Payment Methods

Payment Method Configuration

Admin Setup Path

WooCommerce Point Of Sale → Settings → Payment Methods

Adding New Payment Method

Steps:

  1. Navigate to Payment Methods section
  2. Click "Add New" button
  3. Enter payment details:
    • Payment Name
    • Slug
    • Description
  4. Enable/Disable the payment method
  5. Save changes

Important Note: Added payment methods must be selected in respective outlets to be visible at POS checkout.

Edit/Delete Payment Methods

  • Admin can edit existing payment methods
  • Admin can delete payment methods
  • Changes reflect immediately at assigned outlets

Outlet-Level Payment Assignment

Configuration Steps

Outlets → Edit Outlet → Payment Methods
↓
Select available payment methods for this outlet
↓
Save outlet settings
↓
Methods visible at POS checkout

Payment Processing Features

Split Payment Support

  • Cash + Card: Customers can split payment between cash and card
  • Multiple Methods: Pay using combination of any enabled payment methods
  • Flexible Amounts: Divide total amount across payment methods
  • Transaction Recording: All payment methods recorded separately

Example:

Order Total: $100
Payment 1: $60 (Cash)
Payment 2: $40 (Card via Square Terminal)
Total: $100 (Split Payment)

Payment at Checkout

Cash Payment

  • POS agent accepts cash directly
  • Manual cash drawer management
  • Cash recorded in drawer reports
  • Opening/closing balance tracking

Card Payment (Terminal)

Process:

  1. Select "Other Payment" at checkout
  2. Choose payment terminal (Sunmi/Square)
  3. Customer swipes/inserts card on terminal
  4. Transaction processed externally
  5. POS records payment method used
  6. Receipt generated

Transaction Recording

What POS Records

  • ✅ Payment method used
  • ✅ Transaction amount
  • ✅ Transaction time
  • ✅ Order ID
  • ✅ Customer details
  • ✅ Payment status

Payment Method Display Options

Three View Types Available

1. Grid View Listing

  • Payment methods in normal grid layout
  • Visual card-style display
  • Easy selection for POS agents

Settings Path:

Point of Sale → Settings → Appearance → POS UI Modes → Payment Method Listing

2. Grid With Currency Listing

  • Grid view with currency icons
  • Visual representation of payment types
  • Enhanced user experience
  • Quick identification

3. Dropdown Listing

  • Compact dropdown menu
  • Space-saving design
  • Clean interface
  • Quick selection

Customization

Admin can select preferred view type from:

Settings → Appearance → Payment Method listing

Cash Drawer Integration

Payment Terminal Connection

Many payment terminals support cash drawer connection:

Hardware Setup

Payment Terminal (Square/Sunmi)
↓ (Connected via cable)
Cash Drawer
↓
Automatic drawer opening on cash payment

Drawer Management with Payments

Opening Drawer

  • Automatic popup for opening amount
  • Manual entry of starting cash
  • Enable/disable in settings

Configuration:

Settings → POS Panel Configurations → Opening Amount Drawer Status

Closing Drawer

Tracks:

  • Opening amount
  • Total cash payments received
  • Total card payments (separate)
  • Expected vs actual cash
  • Discrepancies

Payment Reports

Available in Drawer Reports:

  • Cash payments total
  • Card payments total
  • Other payment methods total
  • Payment method breakdown
  • Daily/weekly/monthly summaries

Payment Reports & Analytics

Available Reports

Cash Payment Reports

  • Total cash received
  • Cash payment count
  • Average cash transaction
  • Daily cash summary

Card Payment Reports (Terminal)

  • Total card payments
  • Card transaction count
  • Average card transaction
  • Terminal-wise breakdown

Other Payment Reports

  • PayPal transactions
  • Custom payment methods
  • Split payment statistics
  • Payment method trends
