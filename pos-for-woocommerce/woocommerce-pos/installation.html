WooCommerce Point of Sale - Installation & Setup Guide

Prerequisites

Essential Requirements

Before installation, ensure you have:

RequirementSpecification
WordPressVersion 5.0 or higher
WooCommerceVersion 4.0 or higher
PHPVersion 7.4 or higher (8.0+ recommended)
MySQL5.6+ or MariaDB 10.1+
Memory LimitMinimum 256MB (512MB recommended)
WooCommerce SetupProducts, payment gateways, and shipping configured
User PermissionsAdministrator access to WordPress

Server Requirements

  • SSL Certificate: Recommended for secure transactions
  • Cron Job Access: For scheduled synchronization
  • Node.js: Version 14+ (for full WhatsApp/Node features)
  • File Permissions: Write access to WordPress directories
  • API Access: REST API must be enabled

What You'll Need

✅ Purchase Code from Webkul ✅ WooCommerce Store with products ✅ Admin Credentials for WordPress ✅ Barcode Scanner (optional but recommended) ✅ Thermal Printer (optional for receipts)

Step-by-Step Installation

Method 1: WordPress Admin Dashboard (Recommended)

Step 1: Access Plugin Installation

  1. Log in to your WordPress Admin Panel
  2. Navigate to: Plugins → Add New
  3. Click "Upload Plugin" button at top

Step 2: Upload the Plugin

  1. Click "Choose File" button
  2. Select the downloaded woo-point-of-sale.zip file
  3. Click "Install Now"

Step 3: Activate the Plugin

  1. After successful installation, click "Activate Plugin"
  2. Wait for activation confirmation message

Step 4: Verify Installation

  1. Check WordPress menu for "WooCommerce Point Of Sale"
  2. Verify no error messages appear

Method 2: FTP Installation (Alternative)

Step 1: Prepare Files

  1. Unzip the downloaded package
  2. Extract the woo-point-of-sale folder

Step 2: Upload via FTP

  1. Connect to your server via FTP/SFTP
  2. Navigate to: /wp-content/plugins/
  3. Upload the woo-point-of-sale folder

Step 3: Activate

  1. Go to WordPress Admin → Plugins
  2. Find "WooCommerce Point of Sale"
  3. Click "Activate"

Post-Installation Checks

✅ Clear WordPress cache if using caching plugins ✅ Check for PHP errors in debug log ✅ Verify plugin appears in installed plugins list ✅ Ensure no conflicts with other plugins

- Set opening stock

License Activation

Activation Process

Step 1: Locate Purchase Code

  • Find purchase code in your Webkul purchase email
  • Or access from Webkul client area

Step 2: Enter License Details

  1. Navigate to: Point of Sale → Settings → License
  2. Enter Purchase Code
  3. Enter Registered Email
  4. Click "Activate License"

Step 3: Verify Activation

  • Green checkmark indicates successful activation
  • License details will display expiration date
  • Activation status shows in plugin header

License Management

  • View License: Check status and expiration
  • Renew License: Before expiration for updates
  • Transfer License: Move to another domain
  • Deactivate: Free up license for reuse

Cron Job Setup

Why Cron Jobs Are Important

  • Auto-sync offline orders
  • Scheduled data synchronization
  • Automatic drawer closure
  • Background report generation

Method 1: WordPress Cron (WP-Cron)

For small stores with regular traffic:

Configuration:

  1. Go to: Point of Sale → Settings → General
  2. Find "Cron Type" setting
  3. Select "WP Cron"
  4. Save changes

Note: WP-Cron triggers when visitors access your site. For stores with irregular traffic, use Server Cron.

Method 2: Server Cron (Recommended)

For reliable scheduling:

Step 1: Get Cron Command

  1. Navigate to: Point of Sale → Settings → General
  2. Copy the server cron command:
wget -q -O - https://yourdomain.com/wp-content/plugins/woo-point-of-sale/wcpos-cron.php >/dev/null 2>&1

Step 2: Set Up Crontab

  1. Access server via SSH
  2. Edit crontab: crontab -e
  3. Add line (runs every 5 minutes):
*/5 * * * * wget -q -O - https://yourdomain.com/wp-content/plugins/woo-point-of-sale/wcpos-cron.php >/dev/null 2>&1

Alternative PHP Command:

*/5 * * * * /usr/bin/php -q /path/to/your/wordpress/wp-content/plugins/woo-point-of-sale/wcpos-cron.php >/dev/null 2>&1

Cron Testing & Verification

Test Cron Job:

  1. Manually run cron command
  2. Check for errors in output
  3. Verify sync occurs in POS

Monitor Cron Activity:

  1. Check "Point of Sale → Orders → Offline Sales"
  2. Look for automatic sync timestamps
  3. Review server error logs for cron issues

Node Server Configuration

Prerequisites for Node Server

  • Node.js version 14 or higher installed on server
  • npm or yarn package manager
  • Server access (SSH or control panel)
  • SSL Certificate for HTTPS (recommended)

Step 1: Install Node.js

On Ubuntu/Debian:

sudo apt update
sudo apt install nodejs npm
node --version  # Verify installation

On CentOS/RHEL:

sudo yum install nodejs npm

On cPanel/Shared Hosting:

  • Check if Node.js is available in cPanel
  • Or request installation from hosting provider

Step 2: Configure Node Server in WordPress

Access Node Settings:

Navigate to: Point of Sale → Settings → Node Server

Essential Configuration:

Node Server (Domain/IP) Address: [your-domain.com]
Node Server Port: [3000 or available port]
Node Server HTTPS Status: [Enable if SSL installed]
Node Server Timeout: [30000 ms recommended]

Step 3: Start Node Server

Manual Start via WordPress:

  1. In Node Server settings
  2. Click "Start Server" button
  3. Wait for confirmation message
  4. Status should show "Running"

Manual Start via Command Line:

cd /path/to/wordpress/wp-content/plugins/woo-point-of-sale/node
npm install
node server.js
