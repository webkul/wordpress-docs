WooCommerce POS - Troubleshooting, Compatibility & FAQ Guide

Common Troubleshooting

Page Not Found Error (404)

Issue:

  • Whenever missing page in WooCommerce POS, sales agent sees 404 page not found error

Solutions:

  1. Flush Permalinks:

    • Go to WordPress Dashboard
    • Navigate to Settings > Permalinks
    • Click Save Changes (no changes needed)
    • This refreshes permalink structure

  2. Verify POS Installation:

    • Check plugin is activated
    • Ensure proper installation
    • Verify no file permission issues

  3. Clear Cache:

    • Clear WordPress cache
    • Clear browser cache
    • Clear CDN cache if applicable

  4. Check .htaccess File:

    • Verify .htaccess file exists
    • Ensure correct rewrite rules
    • Restore default if corrupted

Sync Issues

Issue:

  • Offline orders not synchronizing with online store

Solutions:

  1. Manual Sync:

    • Admin can click offline sync button to synchronize offline orders
    • Navigate to POS > Orders > Offline Sales
    • Click Sync Orders button

  2. Auto-Sync Configuration:

    • Enable auto-sync in settings
    • Verify cron jobs running
    • Check network connectivity

  3. Check Sync Queue:

    • Review pending orders
    • Clear sync queue if stuck
    • Restart sync process

  4. Verify Internet Connection:

    • Ensure stable connection
    • Check firewall settings
    • Verify server accessibility

Order Status & Stock Updates

Issue:

  • Stock not updating after order placed

Solution:

  • Order Status: POS system updates product stock when order status updated
  • Ensure order status changes properly
  • Verify stock management enabled
  • Check for conflicts with other plugins

Stock Update Process:

  1. Order placed in POS
  2. Order status changes
  3. Stock automatically updated
  4. Inventory synchronized

Connection Issues

Issue:

  • Node server fails to start or connect

Solutions:

  1. Check Server Connectivity:

    • Verify Node.js installed correctly
    • Check server port availability
    • Ensure firewall allows connections

  2. Verify Configuration:

    • Check Node server settings in admin
    • Verify domain/IP address correct
    • Ensure correct port number

  3. Restart Node Server:

    • Stop running Node server
    • Clear any stuck processes
    • Restart server via PM2 or command line

  4. Check Logs:

    • Review Node server logs
    • Check WordPress error logs
    • Identify specific error messages

Installation Problems

Issue:

  • Plugin fails to install or activate

Solutions:

  1. Verify Correct File:

    • Ensure correct zip file uploaded
    • Check file not corrupted
    • Verify downloaded complete package

  2. Check Permissions:

    • WordPress permissions set properly
    • wp-content/plugins folder writable
    • Correct file ownership

  3. Server Requirements:

    • Verify PHP version (7.4+ required)
    • Check memory limit (256MB minimum)
    • Ensure all dependencies met

  4. Manual Installation:

    • Use FTP method if upload fails
    • Extract and upload via FTP
    • Activate from plugins page

Frequently Asked Questions (FAQ)

Offline Mode

Q: How does the POS system work with offline mode?

A: In offline mode all data is already loaded in POS, you can manage orders with offline mode. Whenever POS connects with the internet all the data or orders will be synchronized with the admin panel automatically.

Key Points:

  • Pre-loaded product data
  • Offline order processing
  • Automatic synchronization when online
  • No data loss
  • Seamless transition

Hardware Integration

Q: Does the POS integrate with the Bluetooth printer, and barcode reader?

A: The hardware connectivity depends on the device. If the device supports the Bluetooth printer, barcode reader, and RFID card reader, then the POS will also support the same.

Supported Hardware:

  • USB devices (printers, scanners)
  • Bluetooth devices (if device supports)
  • RFID card readers
  • Cash drawers
  • Customer displays

Requirements:

  • Device must support hardware
  • Proper drivers installed
  • Compatible operating system
  • Web browser hardware access

Multisite Support

Q: Does it support Multisite?

A: Yes, the latest version of the POS supports WordPress Multisite. It can be used across multiple sites within one network, allowing each site to manage its own POS setup, products, and orders.

Multisite Features:

  • Network-wide activation
  • Site-specific settings
  • Independent outlets per site
  • Centralized or distributed management

Dynamic Themes

Q: Does it support Dynamic Themes?

A: Yes, the POS supports Dynamic Theme customization. Admins can change the POS appearance from Admin Dashboard POS Settings Appearance by updating the primary color, secondary color, and primary light color.

Customization Options:

  • Color schemes
  • Brand colors
  • Light/dark modes
  • Custom styling

Dynamic Invoicing

Q: Does it support Dynamic Invoicing?

A: Yes, the POS supports Dynamic Invoicing. The admin can customize the invoice layout and branding from the admin panel, and the same format is used for all POS orders.

Invoice Customization:

  • Custom templates
  • Branding elements
  • Logo placement
  • Field customization
  • Multiple templates

WhatsApp Notifications

Q: Does it support WhatsApp Notifications?

A: Yes, the POS supports WhatsApp notifications. Automated messages can be sent to customers for order confirmation, invoice creation, and order updates.

WhatsApp Features:

  • Order confirmations
  • Receipt delivery
  • Broadcast messages
  • Personalized messages
  • Greeting messages
