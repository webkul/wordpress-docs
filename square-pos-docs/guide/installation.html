Installation Guide

Setting up the WooCommerce POS Square Reader Native SDK Integration consists of a few simple steps.

Prerequisites

Before starting, ensure you have the following installed:

  1. WordPress CMS: Version 6.5 or higher.
  2. WooCommerce: Version 9.6 or higher.
  3. Point of Sale System for WooCommerce: Version 6.0 or higher.
  4. Square Account: An active Square developer account for API credentials.

1. Manual Plugin Installation

Follow the steps below to install the plugin manually via the WordPress admin panel:

  1. Download: Obtain the woocommerce-pos-square-reader-native-sdk.zip file from your Webkul account.
  2. Login: Log in to your WordPress Admin Panel.
  3. Navigate: Go to Plugins > Add New Plugin in the left-hand sidebar.

  1. Upload: Click the Upload Plugin button at the top of the page.
  2. Choose File: Click Choose File and select the ZIP file from your local storage.
  3. Install: Click Install Now.

  1. Activate: Once the installation is complete, click the Activate Plugin button.

2. Square Developer App Setup

Before using the Square Payment SDK with your POS, you need to set up a Square Developer application:

  1. Sign Up: Create a Square Developer account if you don't have one.
  2. Create Application: In the Square Developer Dashboard, click the + icon to create a new application.
  3. Name Your App: Give it a descriptive name (e.g., "WooCommerce POS Integration").
  4. Copy Credentials: Note down your Application ID, Access Token, and Location ID for the next step.

PRO TIP

Use Sandbox Credentials during development to test payments without processing real transactions.

3. Verification

After activation, you should see a new menu or settings section for POS Square Payment Terminal under your WooCommerce settings.

IMPORTANT

Always verify that you have the latest version of the core Point of Sale System for WooCommerce to avoid compatibility issues.
