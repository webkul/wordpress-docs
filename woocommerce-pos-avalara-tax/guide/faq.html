Frequently Asked Questions

Find answers to common questions about the WooCommerce POS Avalara Tax Integration plugin.

What happens if the internet connection is lost or the Avalara API is down?

To prevent lockouts at the checkout lane, the plugin features an automatic failover fallback. If the connection to the Avalara servers cannot be established, the system automatically reverts to standard WooCommerce tax table configurations to process the transaction. Once the connection is restored, subsequent checkouts will resume querying the Avalara AvaTax API.

Are offline POS sales supported?

While WooCommerce POS supports offline checkout, real-time Avalara AvaTax calculations require an active internet connection. If the register is offline, the POS interface will fall back to using locally stored tax rules. Once the register comes back online and synchronizes the transaction database, the sync process will submit the order logs to WooCommerce and Avalara.

How do I configure tax exemptions for commercial, government, or wholesale customers?

Tax exemption profiles are managed through WooCommerce customer profiles:

  1. Log in to your WordPress dashboard.
  2. Go to Users (or Customers) and edit the target customer's profile.
  3. Fill in the Tax Exemption Code or Exemption ID fields.
  4. On the POS frontend, search for and select this customer for the cart. The plugin automatically sends the exemption payload to Avalara, applying the appropriate tax-free status.

Can I run a multi-outlet store where different registers are in different states or tax zones?

Yes. The plugin determines the tax origin and destination based on the specific POS Outlet address associated with the logged-in sales agent. As long as your outlets are configured with the correct physical address under Point of Sale → Outlets, each register will calculate correct local rates independently.

Why am I seeing a "Calculating tax..." loading state in the POS checkout cart?

The plugin shows a brief loading animation in the tax line during calculations to inform the sales agent that it is actively communicating with the Avalara server. If the loader persists for more than a few seconds, it typically indicates a delay in server response or a configuration discrepancy. In this case, you can click the manual Calculate Tax button to re-trigger the call.

How are shipping charges taxed on POS order fulfillment?

Because Point of Sale transactions typically involve physical pick-ups, shipping tax is not calculated for in-store purchases. However, if you are utilizing the POS interface to create a home-delivery order, the shipping address entered will be sent as the destination address to AvaTax, ensuring shipping tax rules are applied correctly.
