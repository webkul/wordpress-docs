Privacy & GDPR

Full transparency on what data this plugin stores and shares. Shareable with your legal team.

Does This Plugin Set Cookies?

No. The plugin itself sets zero cookies. Cloudflare Turnstile also sets zero cookies — it uses passive browser signals only.

Does It Track Users Across Sites?

No. Neither this plugin nor Cloudflare Turnstile uses cross-site tracking.

What Data Leaves Your Server?

On every form submission with a completed Turnstile challenge, your server sends this to Cloudflare's verify API:

POST https://challenges.cloudflare.com/turnstile/v0/siteverify
  secret=YOUR_SECRET_KEY
  response=TURNSTILE_TOKEN
  remoteip=USER_IP   (optional, sent by default)

That is the only outbound data flow from your server to Cloudflare.

What Data Is Stored in Your WordPress Database?

Two categories: settings and logs.

Settings (permanent)

Standard WordPress options under wp_options:

OptionWhat It Stores
wkcft_site_keyYour Cloudflare Site Key (public)
wkcft_secret_keyYour Cloudflare Secret Key (private)
wkcft_* (many)All your tab settings — no user data

No PII.

Logs (in wp_wkcft_log table)

One row per CAPTCHA validation:

ColumnContentPII?
idRow IDNo
formForm slug (e.g., checkout)No
passed0 or 1No
error_codeCloudflare error code or emptyNo
ipRaw IP address of visitorYes — limited PII
ua_hashMD5 hash of User-AgentNo (hash)
created_atTimestampNo

Why Store IP?

To enable:

  • Top blocked IPs on the Analytics dashboard (so you can one-click blacklist)
  • IP-based rate-limiting (count failures per IP)
  • IP whitelist / blacklist matching

Without IP, you cannot block specific abusers.

Retention

Default retention: 90 days for log rows.

A daily WP-Cron job (wkcft_logger_purge) deletes rows older than 90 days automatically.

To change retention:

add_filter('wkcft_log_retention_days', function() {
    return 30; // keep 30 days instead
});

Manual Purge

DELETE FROM wp_wkcft_log WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);

Rate-Limit Transients (Short-Lived)

For rate limiting, the plugin uses WordPress transients:

TransientContentTTL
wkcft_rl_{md5(IP)}Failed-attempt counter5 min (configurable)
wkcft_fail_{md5(IP)}After-N-failures counter30 min (configurable)
wkcft_used_{md5(token)}Replay protection marker5 min

IPs in transients are hashed (MD5) — the raw IP is not in the transient key.

Transients auto-expire. No manual cleanup needed.

Data Sent to Third Parties

Cloudflare

  • Turnstile token (anonymous, one-time-use)
  • Your Secret Key
  • Optionally: visitor's IP (remoteip param)

Cloudflare uses this ONLY to validate the token. No storage, no training, no advertising use. Cloudflare Privacy Policy: cloudflare.com/privacypolicy.

Webkul (plugin vendor)

  • Zero telemetry sent back to Webkul
  • No analytics, no phone-home
  • The plugin never contacts Webkul servers except for update checks (optional, can be disabled)

GDPR Position

Turnstile and this plugin are designed to be GDPR-compliant by default:

GDPR RequirementStatus
No consent needed for cookies✓ (no cookies)
Data minimization✓ (only IP + form slug + result)
Right to erasurePartial — raw IPs in logs until 90d retention purge
Right to access✓ (admin can view via Analytics)
Data portability✓ (CSV export of logs)
Storage limitation✓ (auto-purge 90d)

If you need stricter GDPR compliance:

  1. Reduce retention via filter to 30 days
  2. Or disable logging entirely:
add_filter('wkcft_log_enabled', '__return_false');
  1. Update your privacy policy to mention:
    • You use Cloudflare Turnstile for bot protection
    • Anonymous tokens and visitor IPs are sent to Cloudflare for validation
    • Link to Cloudflare privacy policy

Sample Privacy Policy Text

For your own privacy policy:

Bot protection

This site uses Cloudflare Turnstile to block automated submissions (spam, bot attacks) on our forms. Turnstile runs in your browser and validates your interactions using anonymous browser signals. No tracking cookies are set.

On form submissions, we send your IP address and an anonymous Turnstile token to Cloudflare for verification. We also log the result (pass/fail) and your IP in our database for up to 90 days for rate-limiting and abuse analysis.

Cloudflare's privacy policy: https://www.cloudflare.com/privacypolicy/

You can opt out by not submitting the form.

Right to Erasure (Customer Requests)

If a customer requests deletion of their data under GDPR:

  1. Identify their IP (if known) — they can tell you, or you can find it from your WC order data
  2. Delete log rows:
DELETE FROM wp_wkcft_log WHERE ip = 'CUSTOMER_IP';
  1. Delete any rate-limit transient (usually already expired):
DELETE FROM wp_options WHERE option_name LIKE '_transient_wkcft_rl_{md5_of_ip}%';

Security of the Secret Key

Your Cloudflare Secret Key is stored in wp_options.wkcft_secret_key. If your WordPress database is compromised, the Secret Key could be stolen.

Defenses:

  • Keep WP core / plugins / theme updated
  • Use strong admin passwords and 2FA
  • Restrict DB access at the server level
  • If you suspect a leak: rotate the Secret Key immediately in Cloudflare, update in plugin

Visitor IP Detection

The plugin resolves the real visitor IP even behind Cloudflare's proxy:

  1. Reads HTTP_CF_CONNECTING_IP (set by Cloudflare)
  2. Falls back to HTTP_X_FORWARDED_FOR (first IP if multiple)
  3. Falls back to REMOTE_ADDR (direct connection)

See WKCFT_Plugin::wkcft_get_real_ip().

User-Agent Handling

User-Agent strings are hashed with MD5 before storage. The plugin never stores the raw UA. This is intentional — UA strings can be fingerprinting vectors.

The hash is used only for:

  • Detecting when the same visitor is making many requests
  • Grouping log entries by client type in analytics

Cache-Control on Admin Pages

The plugin's admin pages are not public, but for good measure they send:

Cache-Control: private, max-age=0

So admin analytics never ends up in a caching layer.

Third-Party Services

The only third-party service this plugin contacts:

ServicePurpose
Cloudflare Turnstilechallenges.cloudflare.com — widget rendering + token verification

No Google Analytics, no Facebook Pixel, no Webkul telemetry.

Compliance With Other Regions

  • CCPA (California) — No sale of personal data, no covered PII collected
  • LGPD (Brazil) — No PII beyond IP, minimal retention, aligned with Brazilian data protection
  • PIPEDA (Canada) — Purpose is clearly defined (bot protection), consent implicit via form submission
