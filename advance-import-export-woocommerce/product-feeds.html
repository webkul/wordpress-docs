Product Feeds

A product feed is an export file formatted for ad networks and marketplaces. Upload it to Google Shopping, Facebook Ads, Amazon, or eBay to list your products.

The plugin has built-in templates for all major networks.

Available Feeds

FeedFormatNetwork
Google ShoppingXMLGoogle Merchant Center
Facebook CatalogCSVFacebook / Instagram Ads
Amazon SellerTSVAmazon Seller Central
eBay File ExchangeCSVeBay
Pinterest ShoppingCSVPinterest Shopping
Bing ShoppingCSVMicrosoft Advertising
PriceGrabberCSVPriceGrabber
ShopzillaCSVShopzilla
Custom FeedAnyYour own template

How Feeds Work

  1. Pick a feed template in the export job
  2. The plugin formats product data per the network's requirements
  3. The file uploads to your server (or any destination)
  4. You give the URL to the ad network
  5. The network reads your products automatically

Feeds are configured in a saved export job — pick the template in the Export tab, destination in the Destination tab.

Google Shopping

Required for Google Merchant Center.

Required Fields

  • id — Your product SKU
  • title — Product name (150 char max)
  • description — Product description
  • link — Full product URL
  • image_link — Main image URL
  • availability — in stock, out of stock, preorder
  • price — Price with currency code (e.g., 29.99 USD)
  • brand — Brand name (or store name)
  • gtin or mpn — Product ID
  • condition — new, used, refurbished

Our Feed Does This Automatically

When you pick the Google Shopping feed, we:

  • Use your SKU as id
  • Use your product name (truncated to 150 chars)
  • Use your description
  • Generate the full product URL
  • Use the featured image
  • Set availability based on stock status
  • Format the price with USD (change in Settings)
  • Use your brand (from WooCommerce Brands or a custom column)
  • Look for gtin in custom fields (add one if missing)
  • Default condition to new (override via a column)

How to Set Up

  1. Scheduled Jobs → Add New → Export
  2. Entity: Products
  3. Format: XML
  4. Feed Template: Google Shopping
  5. Destination: Server Path (to make the URL public) or Amazon S3
  6. Schedule: Daily
  7. Save

Submit to Google

  1. Go to merchants.google.com
  2. Products → Feeds → Add new
  3. Paste your feed URL
  4. Set refresh schedule (match your job schedule)

Facebook Catalog

For Facebook and Instagram Shopping.

Required Fields

  • id, title, description, image_link, link, availability, price, brand, condition

Our Feed Does This

Same as Google, but CSV format with Facebook-specific columns.

Submit

  1. Go to business.facebook.com
  2. Commerce Manager → Catalogs → Add Data Source
  3. Use "Scheduled Feed"
  4. Paste the URL

Amazon Seller

For Amazon Seller Central bulk upload.

Amazon Flat File

Amazon uses tab-separated files (TSV) with very specific columns. Pick the right template for your product category.

Our plugin supports the main "Offer" and "Inventory" formats. For category-specific templates (Electronics, Clothing, etc.), customize the mapping in Step 2.

Submit

  1. Seller Central → Inventory → Add Products via Upload
  2. Download the template for your category
  3. Compare columns with our output
  4. Upload

eBay

eBay File Exchange CSV.

Required Fields

  • Action (always Add, Revise, End)
  • Item ID (for revise/end)
  • Title
  • Description
  • StartPrice
  • Quantity
  • CategoryID

Notes

eBay has hundreds of CategoryID values. Map your WooCommerce category to an eBay category ID using the transformation → Value Mapping.

Pinterest Shopping

Same format as Google Shopping (Pinterest uses Google's spec).

Submit via Pinterest Business Account → Shopping → Add Catalog.

Bing Shopping

Also uses Google Shopping format. Microsoft Advertising reads the same XML.

PriceGrabber and Shopzilla

Older comparison shopping engines. Both use CSV formats slightly different from Google.

Pick the preset, customize columns if needed, and submit the URL to the network.

Custom Feed

If your ad network needs a different format, build a custom template.

How to Create

  1. Export a test file
  2. Note the exact column order and names the network wants
  3. In the Quick Export → Map columns manually
  4. Save the mapping as a preset
  5. Use the preset for future exports

Template Variables

Use these in your field values:

  • {price} — Regular price
  • {sale_price} — Sale price (or empty)
  • {currency} — Currency code from store
  • {product_url} — Full product URL
  • {image_url} — Featured image URL
  • {gallery} — Pipe-separated gallery URLs
  • {categories} — Primary category path

Recommended Schedule

Feeds live as saved jobs — see the last run and next scheduled run at a glance.

Most ad networks refresh their cache every 12-24 hours. Match your export schedule to that:

NetworkRecommended
Google ShoppingDaily at 2 AM
Facebook CatalogDaily
AmazonEvery 4-6 hours (stock-sensitive)
eBayDaily
PinterestDaily
BingDaily

Troubleshooting

ProblemFix
Google rejects feedCheck for required fields. Use Merchant Center diagnostics page
Amazon says "variation mismatch"Variations need specific fields (color, size, parent SKU)
Feed URL gives 404File was auto-cleaned. Increase retention in Settings
Prices show wrong currencySet Store Currency in WooCommerce → Settings → General
Out-of-stock products in feedSet feed filter to "In Stock Only" in the job
Image URLs not accessibleMake sure Media Library URLs are public
