Export Destinations

After an export runs, the file can be sent to any of these destinations automatically.

At a Glance

DestinationWhat It Does
File DownloadDownload in your browser
Server PathSave to a folder on the server
EmailSend as email attachment
FTPPush to an FTP server
SFTPSecure FTP push
Google SheetsWrite rows to a Google Sheet
Google DriveUpload to Drive
DropboxUpload to Dropbox
Amazon S3Upload to S3 bucket
REST APIPOST to any endpoint
WooCommerce REST APIPush to another WC store

File Download

The default destination. After export, click the download link in the result screen.

How to Use

  1. Quick Export → Pick entity → Pick format → Click Run Export
  2. Click Download on the result

Files are stored on your server for 24 hours, then auto-cleaned.

Server Path

Save to a folder on the server instead of downloading.

How to Use

  1. Create a scheduled job (exports only)
  2. Tab: Destination
  3. Pick Server Path
  4. Enter the path (e.g., /wp-content/uploads/exports/)

Useful for backups that stay on the server.

Email

Send the exported file as an email attachment.

How to Use

  1. In a scheduled job → Destination → Email
  2. Fill in:
    • To
    • CC (optional)
    • BCC (optional)
    • Subject
    • Body (HTML)
  3. Save

Template Variables

You can use these in Subject and Body:

  • {job_title}
  • {entity_type}
  • {row_count}
  • {file_size}
  • {timestamp}

File Size Limit

Email has a 10MB attachment limit by default. For bigger exports, use FTP or cloud storage.

FTP

Push to an FTP server.

FTP panel — host, port, credentials, remote path, plus a Test Connection button.

How to Use

  1. Destination → FTP
  2. Fill in:
    • Host
    • Port (21 default)
    • Username
    • Password
    • Remote path (where to save)
  3. Click Test Connection to verify
  4. Save

Security Warning

FTP sends credentials in plain text. Use SFTP instead whenever possible.

SFTP

Secure FTP with SSH.

How to Use

Same as FTP, but:

  • Port 22 default
  • Can use SSH key instead of password

SSH Key Auth

  1. Generate an SSH key on your server (or use an existing one)
  2. Add the public key to the SFTP server's authorized_keys
  3. In the job, leave Password blank and fill Key Path

Google Sheets

Write rows directly to a Google Sheet.

Google Sheets panel — spreadsheet URL, sheet name, and clear-before-write option.

How to Use

  1. Set up Google API in Settings → Google APIs (need OAuth, not just API Key)
  2. Authorize the plugin via OAuth flow
  3. In the job, Destination → Google Sheets
  4. Fill in:
    • Spreadsheet URL or ID
    • Sheet name
    • Clear sheet before writing (yes/no)
  5. Save

Use Cases

  • Share product catalog with a team
  • Live dashboard in Google Sheets
  • Make sheet the source of truth

Google Drive

Upload the export file to Google Drive.

How to Use

  1. Settings → Google APIs → OAuth credentials
  2. Job → Destination → Google Drive
  3. Pick a folder (or leave blank for root)
  4. Filename pattern (can use variables)

Dropbox

Dropbox panel — folder path and filename pattern for the upload.

How to Use

  1. Settings → Dropbox → Access Token
  2. Job → Destination → Dropbox
  3. Enter folder path (e.g., /Exports/)
  4. Filename pattern

Amazon S3

Upload to S3 bucket.

Amazon S3 panel — bucket, path prefix, region, and optional per-job access keys.

How to Use

  1. Set up AWS credentials in Settings → Amazon S3
  2. Required IAM permissions:
    • s3:PutObject
    • s3:ListBucket
  3. Job → Destination → Amazon S3
  4. Fill in:
    • Bucket
    • Path prefix (e.g., exports/)
    • Region
    • Access Key (optional if set in Settings)
    • Secret Key (optional)

Use Cases

  • Enterprise backups
  • Cheap long-term storage
  • CDN-fronted downloads

REST API

POST the export data to any endpoint.

How to Use

  1. Job → Destination → REST API
  2. Fill in:
    • Endpoint URL
    • Method (POST, PUT, PATCH)
    • Headers (JSON)
    • Auth (none, basic, bearer, custom)
  3. Save

The plugin sends the export file as the request body (or as a form upload).

WooCommerce REST API

Push to another WooCommerce store.

How to Use

  1. On the target store: WooCommerce → Settings → Advanced → REST API → Add Key with Write permission
  2. Copy Consumer Key and Secret
  3. On your store: Job → Destination → WooCommerce REST API
  4. Fill in:
    • Target store URL
    • Consumer Key
    • Consumer Secret

What It Does

The plugin sends each row via the standard WC REST API. So you can push products from Store A to Store B without any custom code.

Multi-Store Sync

Pair this with delta import on Store A → the two stores stay in sync automatically.

Current Limitations

Important note: All 12 destination handlers are fully implemented in code. However, the connection between the export engine and the destination handlers (a single action hook wiring) is still in final testing. Until the next release:

  • File Download works
  • Other destinations save the file to wp-content/uploads/wkaie-exports/ but do not automatically push to the remote destination

The destination UI is fully functional for configuration.

Filename Patterns

For destinations that need a filename, use these variables:

VariableExample
{entity}products
{format}csv
{date}2026-04-14
{datetime}2026-04-14-103000
{timestamp}1744588800
{count}1000

Example: products-export-{date}.{format} → products-export-2026-04-14.csv

Troubleshooting

ProblemFix
Email not sendingInstall a SMTP plugin like WP Mail SMTP
FTP test passes but upload failsFTP server may block passive mode. Try SFTP
Google Sheets "quota exceeded"Google API has rate limits. Use scheduled export, not live
S3 403 errorCheck IAM policy includes s3:PutObject
Dropbox token expiredDropbox tokens rotate. Regenerate and re-save
REST API 401Auth header is wrong. Test with Postman first
